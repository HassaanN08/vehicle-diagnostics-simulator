# v0.4 Plan

## Current Status

The simulator currently supports:

- ECUs
- DTCs
- fault severity
- adding ECUs
- adding faults
- clearing faults
- scanning vehicle
- diagnostic logs
- ECU communication status
- result enums
- CMake build support

## Next Possible Directions

### Option 1: Unit Tests

Add a basic testing setup to verify:

- duplicate ECU names are rejected
- DTCs can be added
- faults can be cleared
- offline ECUs affect vehicle health
- result enums return correctly

### Option 2: ECU Readiness / Operational State

Add a second state concept separate from communication status:

- Ready
- Not Ready
- Error
- Sleep

### Option 3: Diagnostic Sessions

Introduce simple diagnostic modes:

- Default session
- Extended session

Certain actions could only be allowed in extended session.

### Option 4: Fault History

Store cleared faults separately from active faults.

## Recommended Next Step

The next best step is unit testing.

Reason:

Before adding more behavior, the project needs confidence that existing features still work after refactors.
